# Отчёт по лабораторной работе

## Общая информация

В рамках лабораторной работы был использован проект `https://github.com/danluki/qrgen-lab`, представляющий собой микросервисное веб-приложение для генерации QR-кодов. Для проекта была настроена автоматизация процессов сборки, тестирования и проверок безопасности с использованием GitHub Actions.

## Цель работы

Изучить подходы к встраиванию автоматических проверок безопасности в процесс сборки и проверки программного проекта.

## Используемые инструменты

- GitHub Actions — платформа CI/CD.
- Go — язык разработки backend-сервисов проекта.
- Vue 3 + Vite + Bun — стек frontend-части проекта.
- Docker Compose — запуск и оркестрация сервисов приложения.
- Semgrep — статический анализ исходного кода.
- Gitleaks — поиск секретов в репозитории.
- CodeQL — анализ безопасности кода.

## Выполненная работа

В ходе лабораторной работы для проекта был настроен CI/CD-процесс и добавлены автоматические проверки качества и безопасности.

В процессе выполнения были сделаны следующие действия:

- подключён `Semgrep` для статического анализа кода и поиска небезопасных паттернов;
- подключён `Gitleaks` для поиска секретов в истории и содержимом репозитория;
- создан отдельный workflow `codeql.yml` для анализа безопасности с помощью `CodeQL`;

По итогам настройки репозитория были реализованы автоматические проверки, которые позволяют контролировать корректность сборки проекта, стабильность работы приложения и отсутствие типовых проблем безопасности до слияния изменений в основную ветку.

## Результат

В результате выполнения лабораторной работы для проекта был успешно настроен SecOps процесс. Пайплайн автоматически запускается при изменении кода и при создании pull request, включает автотесты, сборку backend и frontend, проверку Docker Compose-конфигурации, smoke-test приложения, статический анализ исходного кода и поиск секретов в репозитории. Для анализа безопасности используются инструменты `Semgrep`, `Gitleaks` и `CodeQL`.
